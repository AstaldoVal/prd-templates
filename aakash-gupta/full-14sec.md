# Aakash Gupta PRD — ChatPRD Variant (14 sections)

SOURCE: https://www.aakashg.com/how-to-write-product-requirements/
CHATPRD_VARIANT: "Aakash's PRD" (community template)
FETCHED: 2025-03-04

---

## 1. Problem
Short, testable problem statement. Includes customer pain and business value.

## 2. High Level Approach
Short summary of the intended approach, not prescribing the full solution.

## 3. Narrative
- Common use case
- Edge case
- Before/after stories to build empathy

## 4. Goals
- **Measurable** (with baselines and targets)
- **Immeasurable** (how it should feel)

## 5. Metrics
- North Star Metric
- Secondary Metrics
- Guardrail Metrics

## 6. Impact Sizing Model
- Step-by-step calculation with sources
- Revenue/bottom-line impact, confidence, and assumptions

## 7. Non-goals
What's explicitly out of scope and why.

## 8. Solution Alignment
- **What's In**
- **What's Out**
- **Open Questions**

## 9. Key Features: Plan of Record
Feature | Priority | Description | Size estimate  
(one row per feature)

## 10. Future Considerations
- **Planned Follow-ups** (timing/phase)
- **Build-Now Implications**

## 11. Key Flows
Detailed screen-by-screen walkthroughs, including empty and error states.

## 12. Key Logic
Business rules, edge case handling, and scenarios.

## 13. Launch Plan
- **Phases** (purpose, criteria, duration)
- **Experiment Design**
- **Risks & Dependencies**

## 14. Key Milestones
Table of milestones (date, owner, status) and explicit dependencies.
